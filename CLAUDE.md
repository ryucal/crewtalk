## Project Overview

Crewtalk is a Korean transportation/logistics coordination app built with Flutter and Firebase (project ID: `crewtalk8`). It serves drivers, managers, and super-admins with real-time chat, GPS tracking, staff reports, and emergency alerts.

## Common Commands

```bash
# Install dependencies
flutter pub get

# Run app in development
flutter run

# Run code generation (Riverpod annotations, Drift schema)
flutter pub run build_runner build --delete-conflicting-outputs

# Run tests
flutter test

# Run a single test file
flutter test test/some_test.dart

# Build Android APK
flutter build apk

# Build iOS
flutter build ios

# Deploy Firestore security rules
firebase deploy --only firestore:rules

# Deploy Cloud Functions
firebase deploy --only functions

# Deploy Storage rules
firebase deploy --only storage:rules
```

> Run `build_runner` whenever `app_database.dart` schema changes to regenerate `app_database.g.dart`.

## Architecture

### State Management: Riverpod

All providers live in `lib/providers/`. Key providers:

- **`userProvider`** (`StateNotifierProvider`) — authenticated user, login/logout, persisted to SharedPreferences via `UserSessionStorage`
- **`roomProvider`** (`StateNotifierProvider`) — chat room list, merged from Firestore + local state; always prepends virtual rooms 998 and 999
- **`currentRoomProvider`** (`StateProvider`) — the currently open `RoomModel`
- **`roomMessagesStreamProvider`** (`StreamProvider.autoDispose.family<..., int>`) — live message stream for a room ID; returns empty stream for virtual rooms or offline mode
- **`chatMessagesNotifierProvider`** (`StateNotifierProvider.autoDispose.family`) — cursor-based paginated loading; state includes `isInitializing`, `isLoadingMore`, `hasMore`, `hasError`, `restoredFromCache`; initial load 50 msgs, "load more" fetches 50 older; tracks `_oldestDoc`/`newestDoc` for Firestore query cursors
- **`incomingEmergencyProvider`** — holds FCM emergency payload for `GlobalEmergencyOverlay`; only shown to manager/superadmin
- **`companyProvider`**, **`adminMessageProvider`**, **`dbMessageProvider`** — company list, virtual room 999 summaries, virtual room 998 DB search results
- **`gpsActiveProvider`** / **`gpsRunInfoProvider`** — GPS session toggle and current run metadata
- **`calendarItemsProvider`** (`StreamProvider`) — streams calendar/schedule items
- **`emergencyContactsProvider`** (`StreamProvider`) — streams emergency contact list from Firestore
- **`reportCooldownEndProvider`** (`StateProvider`) — rate-limit cooldown timestamp for staff report submissions

### Firebase Integration

- **Auth:** Email/password only. Emails are synthetic (`{digits}@crew.co.kr`) generated by `PhoneAuthUtils.syntheticEmailFromDigits`. Company password validated server-side via `verifyCompanyPassword` Cloud Function.
- **Firestore collections:** `/config/rooms` (room list), `/config/kakao_nav_links`, `/rooms/{roomId}/messages` (chat), `/users/{uid}`, `/tracks/{trackId}/points/{pointId}` (GPS), `/company_profiles/{name}` (readable), `/company_private/{name}` (Functions only), `auth_attempts/{uid}` (rate limiting).
- **Cloud Functions** (`functions/index.js`): `verifyCompanyPassword` (rate-limited to 10 failed attempts per 10-min window per UID), `onEmergencyMessageCreated` (FCM to managers/superadmins + denormalizes `lastMessage`), `onChatMessageCreated` (FCM to room companies, skips 998/999), `adminUpsertCompany`, `adminDeleteCompany`.
- **FCM tokens** stored in `users/{uid}.fcmTokens[]`; push sent in 500-token batches from Functions.
- **Firebase Storage** used for chat image uploads; `ChatImageCache` provides a local disk cache keyed by URL SHA-256 hash.

### Offline/Demo Mode

`AuthRepository.firebaseAvailable` is a static flag checked before every Firebase call. If Firebase init fails, the app falls back to sample data in `utils/sample_data.dart`. All service methods must check this flag.

### Virtual Chat Rooms

Rooms **998** and **999** do not exist in Firestore — they are local-only UI constructs:
- Room **998** — Driver/vehicle database search (`dbMessageProvider`)
- Room **999** — Daily operations summary / Work Hub (`adminMessageProvider`)

`roomMessagesStreamProvider` returns an empty stream for these IDs. `RoomNotifier.remove()` and `.pin()` guard against modifying them.

### Role-Based Access Control

| Role | Capabilities |
|------|-------------|
| `driver` | Chat, GPS tracking, staff reports |
| `manager` | + Virtual admin rooms (998/999), broadcast notices |
| `superadmin` | + Room/company CRUD, config writes, moderation |

Company `"관리자"` is treated as a special admin company in Firestore rules. Role strings are case-normalized in `UserModel`. Key computed properties:
- `isStaffElevated` = manager or superadmin
- `canManageRoomsAndConfig` = superadmin or company '관리자'
- `canBroadcast` = manager or superadmin
- `canModerateChatMessages` = manager or superadmin (`isStaffElevated`)

### Message Type System

Messages have a `type` enum: `text`, `report`, `vendorReport`, `notice`, `image`, `emergency`, `dbResult`, `summary`, `maintenance`. `RoomModel` has a `roomType` enum (`normal`, `vendor`, `maintenance`) used to filter broadcast notices via `noticeForRoomType`. **All serialization/deserialization goes through `lib/services/message_firestore_codec.dart` exclusively.** Each type maps to distinct Firestore fields: `reportData`, `vendorData`, `maintenanceData`, `imageUrls[]` (with legacy `imageUrl`), `emergencyType`, `noticeForRoomType`.

### Real-Time Room Sync

`FirestoreRoomSync` widget (mounted at app root) subscribes to `config/rooms` and `config/kakao_nav_links`, then merges updates into `roomProvider` while preserving local unread counts and kakao nav links.

### Navigation

Go Router (`lib/router/app_router.dart`) with `routerProvider`. Routes: `/login`, `/signup`, `/rooms`, `/chat`, `/profile`. Router refresh triggered via `ValueNotifier<int>` that watches `userProvider` changes.

### GPS Tracking

Uses `flutter_foreground_task` + `geolocator`. Tracks written to `/tracks/{trackId}/points/{pointId}` (trackId = `ownerUid_timestamp`). Points buffered 30s then batch-written (max 450 per batch). Auto-stops after 1 hour.

### Push Notifications

- Emergency → managers/superadmins via `crewtalk_emergency` Android channel (high priority)
- Regular chat → room's associated companies via `crewtalk_messages` channel
- Local display: `FcmPushService` + `flutter_local_notifications`
- Sound: `foreground_notify_sound.dart` via `audioplayers` (`chat_notify.mp3` / `emergency_notify.mp3`)

### Kakao Maps Integration

Room nav links at `/config/kakao_nav_links` with `rooms.{roomId}` arrays. Each entry is a `RoomKakaoNavLink` (id, label, kakaoShareUrl). Write access restricted to superadmin.

### App Initialization Sequence (`lib/main.dart`)

1. Lock orientation to portrait
2. `GpsService.init()` — foreground task registration
3. Firebase init (sets `firebaseAvailable=false` on failure)
4. FCM background handler registration (`@pragma('vm:entry-point')`)
5. Load session user from Firestore; fall back to SharedPreferences
6. FCM token sync for logged-in user
7. Delete Drift `CachedMessages` older than 30 days (fire-and-forget)
8. Run app with `FirestoreRoomSync` at root

### Local Database (Drift/SQLite)

`lib/database/` — Drift SQLite, schema version 2, 4 tables:

| Table | Purpose |
|-------|---------|
| `CachedMessages` | Offline message cache; PK = `firestoreDocId`; full message JSON in `jsonData` |
| `CachedRooms` | Room metadata cache |
| `LocalReadState` | Per-room last-read timestamp |
| `OutboxMessages` | Failed-send queue; retried on network recovery |

`outbox_dao.dart` provides `enqueueOutboxMessage` / `getPendingOutboxMessages` / `dequeueOutboxMessage` for the retry queue. `message_dao.dart` supports cursor-based pagination via `getOlderMessages`.

### Message Bubble Interactions

`lib/widgets/message_bubbles.dart` — long-press on a bubble shows a context menu (edit + delete) gated by:
- **Edit:** `isAdmin && (type is text-editable || type == report)` → text types use `updateMessageText()`; report type uses `_ReportEditDialog`
- **Delete:** `isAdmin || msg.isMe`

`notice` and `emergency` bubble builders (`_buildNotice`, `_buildEmergency`) currently have **no** long-press handler.

### Services & Utilities

- **`ChatFirestoreRepository`** — all Firestore reads/writes for rooms and messages; `updateMessageText`, `updateReportData`, `deleteMessage`, `updateReactions`, `sendMessage`
- **`TrackFirestoreRepository`** — GPS track CRUD; guards on `firebaseAvailable`
- **`ChatImageCache`** — singleton disk cache under `getApplicationSupportDirectory()/chat_image_cache/`; images >450KB stored full-res, previewed at lower res in list
- **`AppColors`** — centralized color constants; role-color palette for time-of-day shifts
- **`PhoneAuthUtils`** — Korean mobile number normalization + synthetic email generation
- **`helpers.dart`** — `outgoingMessageUserId()`, `timeNow()`, `dateToday()`, formatting helpers

### Persistence (`UserSessionStorage`)

SharedPreferences keys:
- `crewtalk_user_session_v1` — serialized user JSON
- `room_last_read_{roomId}` — per-room last-read timestamp
- `chat_cursor_ms_{roomId}` — message pagination cursor per room
