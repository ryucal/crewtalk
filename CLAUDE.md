## Project Overview

Crewtalk is a Korean transportation/logistics coordination app built with Flutter and Firebase (project ID: `crewtalk8`). It serves drivers, managers, and super-admins with real-time chat, GPS tracking, staff reports, and emergency alerts.

## Common Commands

```bash
# Install dependencies
flutter pub get

# Run app in development
flutter run

# Run code generation (Riverpod annotations, etc.)
flutter pub run build_runner build --delete-conflicting-outputs

# Run tests
flutter test

# Run a single test file
flutter test test/some_test.dart

# Build Android APK
flutter build apk

# Build iOS
flutter build ios

# Deploy Firestore security rules
firebase deploy --only firestore:rules

# Deploy Cloud Functions
firebase deploy --only functions

# Deploy Storage rules
firebase deploy --only storage:rules
```

## Architecture

### State Management: Riverpod

All state is managed via Riverpod providers. Key providers:

- **`userProvider`** (`StateNotifierProvider`) — authenticated user, login/logout, persisted to SharedPreferences via `UserSessionStorage`
- **`roomProvider`** (`StateNotifierProvider`) — chat room list, merged from Firestore + local state; includes virtual rooms 998 and 999
- **`currentRoomProvider`** (`StateProvider`) — the currently open `RoomModel`
- **`roomMessagesStreamProvider`** (`StreamProvider.autoDispose.family<..., int>`) — live message stream for a room ID; returns empty stream for virtual rooms or offline mode
- **`incomingEmergencyProvider`** — holds FCM emergency payload for `GlobalEmergencyOverlay`; only shown to manager/superadmin roles
- **`companyProvider`**, **`adminMessageProvider`**, **`dbMessageProvider`** — company list, virtual room 999 summaries, virtual room 998 DB search results
- **`gpsActiveProvider`** / **`gpsRunInfoProvider`** — GPS session toggle and current run metadata (name, car, route, count)
- **`chatMessagesNotifierProvider`** (`StateNotifierProvider.autoDispose.family`) — cursor-based paginated message loading; initial load 50 msgs, "load more" fetches 50 older; tracks `_oldestDoc` and `newestDoc` for query cursors
- **`calendarItemsProvider`** (`StreamProvider`) — streams calendar/schedule items
- **`reportCooldownEndProvider`** (`StateProvider`) — tracks rate-limit cooldown timestamp for staff report submissions

### Firebase Integration

- **Auth:** Email/password only. Emails are synthetic (`{digits}@crew.co.kr`) generated by `PhoneAuthUtils.syntheticEmailFromDigits`. Company password validated server-side via `verifyCompanyPassword` Cloud Function.
- **Firestore collections:** `/config/rooms` (room list), `/config/kakao_nav_links` (Kakao Maps deep links per room), `/rooms/{roomId}/messages` (chat), `/users/{uid}`, `/tracks/{trackId}/points/{pointId}` (GPS), `/company_profiles/{name}` (readable), `/company_private/{name}` (no client access — Functions only), `auth_attempts/{uid}` (rate limiting state for `verifyCompanyPassword`).
- **Cloud Functions** (`functions/index.js`): `verifyCompanyPassword` (reads `config/companies.items[]` for password data; rate-limited to 10 failed attempts per 10-min window per UID via `auth_attempts`), `onEmergencyMessageCreated` (FCM to managers/superadmins; also denormalizes `rooms/{roomId}.lastMessage`), `onChatMessageCreated` (FCM to room companies, excluding sender and muted rooms, skips 998/999), `adminUpsertCompany`, `adminDeleteCompany`.
- **FCM tokens** stored in `users/{uid}.fcmTokens[]`; push sent in 500-token batches from Functions. Local notification sound via `audioplayers`.
- **Firebase Storage** used for chat image uploads; `ChatImageCache` provides a local disk cache keyed by URL hash (images >450KB are stored as full-res originals, previewed at lower resolution in the list).

### Offline/Demo Mode

`AuthRepository.firebaseAvailable` is a static flag checked before every Firebase call. If Firebase init fails (network unavailable), the app falls back to sample data in `utils/sample_data.dart`. All service methods must check this flag to avoid crashing without connectivity.

### Virtual Chat Rooms

Rooms **998** and **999** do not exist in Firestore — they are local-only UI constructs:
- Room **998** — Driver/vehicle database search (`dbMessageProvider`)
- Room **999** — Daily operations summary (`adminMessageProvider`)

`roomMessagesStreamProvider` returns an empty stream for these IDs.

### Role-Based Access Control

| Role | Capabilities |
|------|-------------|
| `driver` | Chat, GPS tracking, staff reports |
| `manager` | + Virtual admin rooms (998/999), broadcast notices |
| `superadmin` | + Room/company CRUD, config writes, moderation |

Company `"관리자"` is treated as a special admin company in Firestore rules. Role strings are case-normalized — the `UserModel` handles variations and legacy `isAdmin` boolean. Key computed properties: `isStaffElevated` = manager or superadmin; `canManageRoomsAndConfig` = superadmin or company '관리자'; `canBroadcast` = manager or superadmin; `canModerateChatMessages` = superadmin.

### Message Type System

Messages have a `type` enum: `text`, `report`, `vendorReport`, `notice`, `image`, `emergency`, `dbResult`, `summary`, `maintenance`. `RoomModel` has a `roomType` enum (`normal`, `vendor`, `maintenance`) used to filter broadcast notices via `noticeForRoomType`. Serialization/deserialization is handled exclusively in `lib/services/message_firestore_codec.dart`. Each type maps to distinct Firestore fields: `reportData` (staffing), `vendorData` (subcontractor), `maintenanceData` (vehicle), `imageUrls[]` (with legacy single `imageUrl`), `emergencyType`, `noticeForRoomType` (optional room-type filter for broadcast notices).

### Real-Time Room Sync

`FirestoreRoomSync` widget (mounted at app root) subscribes to `config/rooms` and merges Firestore updates into `roomProvider`, preserving local unread counts. Each room streams its latest message for the list preview.

### Navigation

Go Router (`lib/router/app_router.dart`) with `routerProvider`. Routes: `/login`, `/signup`, `/rooms`, `/chat`, `/profile`. Auth-redirect logic is embedded in the router: unauthenticated users go to `/login`; authenticated users go to `/rooms`. Router refresh is triggered via a `ValueNotifier<int>` that watches `userProvider` changes.

### GPS Tracking

Uses `flutter_foreground_task` for background operation and `geolocator` for location. GPS tracks written to `/tracks/{trackId}/points/{pointId}` in Firestore (trackId = `ownerUid_timestamp`). Foreground service initialized in `GpsService.init()` at app startup. Points buffered for 30 seconds then batch-written (max 450 per batch due to Firestore limits). Auto-stops after 1 hour. Point fields: lat, lng, accuracy, heading, speed, timestamp.

### Push Notifications

- Emergency messages → managers/superadmins via `crewtalk_emergency` Android channel (high priority)
- Regular chat → room's associated companies via `crewtalk_messages` channel
- Local notification display handled by `FcmPushService` + `flutter_local_notifications`
- Sound playback via `foreground_notify_sound.dart` using `audioplayers`

### Kakao Maps Integration

Room navigation links stored at `/config/kakao_nav_links` with `rooms.{roomId}` arrays. Each entry is a `RoomKakaoNavLink` (id, label, kakaoShareUrl). Write access restricted to superadmin in Firestore rules. `FirestoreRoomSync` watches this collection and merges links into `roomProvider`.

### App Initialization Sequence (`lib/main.dart`)

1. Lock orientation to portrait
2. `GpsService.init()` — foreground task registration
3. Firebase init (sets `firebaseAvailable=false` on failure)
4. FCM background handler registration (`@pragma('vm:entry-point')`)
5. Load session user from Firestore; fall back to SharedPreferences
6. FCM token sync for logged-in user
7. Run app with `FirestoreRoomSync` at root

### Local Database (Drift/SQLite)

`lib/database/` contains a Drift-based local SQLite database for offline caching:
- `app_database.dart` / `app_database.g.dart` (generated) — 3 tables: `CachedMessages`, `CachedRooms`, `LocalReadState`
- `message_dao.dart` — message CRUD; supports cursor-based pagination (`getOlderMessages`) and bulk upsert
- `room_dao.dart` — room cache + per-room read state
- `database_provider.dart` — Riverpod provider exposing the `AppDatabase` singleton

On startup, `main.dart` fires-and-forgets a deletion of `CachedMessages` older than 30 days.

Run `flutter pub run build_runner build --delete-conflicting-outputs` whenever the schema changes to regenerate `app_database.g.dart`.

### Additional Services & Utilities

- **`TrackFirestoreRepository`** (`lib/services/track_firestore_repository.dart`) — GPS track CRUD on `tracks/{trackId}` and sub-collection `points/{autoId}`; all methods guard on `AuthRepository.firebaseAvailable`.
- **`ChatImageCache`** (`lib/services/chat_image_cache.dart`) — Singleton disk cache under `getApplicationSupportDirectory()/chat_image_cache/`. Cache key = SHA-256 of URL. Broadcasts updates via `cacheUpdated` stream. Images >450KB show thumbnail in list, full-res on tap.
- **`FcmPushService`** (`lib/services/fcm_push_service.dart`) — Manages foreground FCM delivery + `flutter_local_notifications` display; initializes Android notification channels.
- **`ForegroundNotifySound`** (`lib/services/foreground_notify_sound.dart`) — Plays `chat_notify.mp3` / `emergency_notify.mp3` from `assets/sounds/` via `audioplayers` for foreground notification sound.

**`lib/utils/`:**
- `AppColors` — Centralized color constants (black primary, `#F5F5F5` background, role-color palette for time-of-day shifts).
- `PhoneAuthUtils` — Korean mobile number normalization and synthetic email generation (`{digits}@crew.co.kr`).
- `helpers.dart` — `outgoingMessageUserId()`, `timeNow()`, `dateToday()`, and other formatting helpers.
- `sample_data.dart` — Offline fallback: 7 sample companies, 5 drivers, 5 vehicles.

**New widgets:**
- `GlobalEmergencyOverlay` — `Positioned.fill` overlay that watches `incomingEmergencyProvider`; renders only when a non-null payload exists; manager/superadmin only.
- `ChatNetworkImage` — Network image widget backed by `ChatImageCache`, falls back gracefully on cache miss.
- `ChatGalleryPhoto` — Full-screen photo viewer using `photo_view` for image messages.

### Persistence (`UserSessionStorage`)

SharedPreferences keys:
- `crewtalk_user_session_v1` — serialized user JSON
- `room_last_read_{roomId}` — per-room last-read timestamp (for unread counts)
- `chat_cursor_ms_{roomId}` — message pagination cursor per room
